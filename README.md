，# Simon_game
掌上游戏，这是一个简单的颜色配对游戏，里面有一个微控制器或微处理器。你按下开始按键，然后游戏会点亮其中的一个面板，然后你按下那个面板，它就会点亮两个面板，你按下两个面板，以此类推，直到你再也记不住为止。
一、游戏玩法 
1. 开机初始化：上电后，游戏会进行一次自检，所有灯闪烁一次，喇叭发出提示音，进入待机状态。
2. 生成序列：游戏开始后，MCU会随机生成一个长度为1的序列（比如：绿色灯闪+对应音效）。
3. 玩家复刻：玩家需要按下和灯/声音对应的按钮，重复这个序列。
4. 难度递增：如果按对了，序列长度+1（比如变成：绿→红），再次播放完整序列让玩家记忆；按错了，游戏结束，播放失败音效并显示最终分数。
5. 挑战极限：序列会越来越长，直到玩家出错为止，记录的最高长度就是你的得分
我们需要能发出声音，发出光，能有输入，我们可以用一个非常小的微控制器来完成这个（以Arduino为基础）。Arduino是一个技术平台，基于一个Atmel小型8位处理器，而Arduino本身并非真正的处理器，也不是真正的开发板，它实际上是你如何编程它，如何理解他，他是一个工具，一个最初用java编译的工具。它在底层隐藏了编译器，包括C编译器和GNU编译器。因此，我们将使用一个兼容Arduino的处理器和一个兼容Arduino的核心设计，这样我们就能用Arduino工具来编程它。如果我们拿一个Arduino，然后只连接四个开关作为按钮，再连接四个LED灯，然后连接一些声音组件，再连接几个开关用于控制这些其他按钮，你会像编程Arduino一样来编程它。这样，通过输入/输出端口，你就可以控制所有这些按钮和灯光了。

USB转串口UART,多年前，所有的个人电脑都配有串口，这些串口都基于RS232标准，RS232是一种非常简单的协议，它有TX和RX两条线，以及一条地线，串口数据会通过TX和RX线进行时钟输出，他是一个非同步协议，因此会以固定速度运行，它的性质是模拟的，电压为正负12V。此外还有数字TTL串口，其电压为＋5伏和0伏，用来表示逻辑上的1和0，这要容易一些，因为原始的RS-232电压会达到+12V，-12V。USB（通用串行总线）问世后，那些简单的接口就被淘汰了。十年来，各公司都推出了USB设备，能够创建标准的数字TTL版串口，所以你可以用一个USB口，把它接到你的设备上，然后设备会输出串口的TX、RX和地线，不过要让它正常工作，你的电脑上需要一个驱动程序，一个帮你实现这一功能的驱动程序，这样你就可以通过操作系统内置的某种API创建一个串口，并发送串口数据，然后从这个连接到USB端口的神奇设备中，它带有一个标准的RS232串口，你就能得到TX和RX信号。现在的问题是，你可以通过USB端口连接电脑和电脑，只要它们有USB接口就行，但如果你有一些像微控制器这样的小设备怎么办？一个基于微控制器的设备，他没有USB接口，他有的是一个串行UART接口，它有TX和RX两条线，以及一条地线。接着一个设备就被开发出来了，这就是所谓的USB转串口桥接器，它的作用就是接收USB信号，然后作为USB设备套件的一部分，有一个串口设备，最终输出就是你能得到TX/RX信号，然后就可以和微控制器连接起来。这样微控制器就能通过TX、RX和串口数据包进行通信，发送ASCLL字符，接着USB转串口桥接器接收到信号，将它转换成USB数据包，再发送回电脑。现在问题来了：在电脑上，我们需要一个能识别这种信号的驱动程序，因此电脑里必须安装一个USB串口驱动程序，你的应用程序与驱动程序进行通信，并且希望这个驱动程序是透明的。因此从你的角度来看，如果你只是创建一个COM端口，那个驱动程序会为你提供一个USB COM端口，因此它是完全透明的。从编程的角度来看，你无需修改代码。假设你在windows等系统中使用了一个API调用来打开一个COM端口，并且指定了一个端口号，那么这个API仍将完全不受影响地工作，因为驱动程序的工作就是去安装自己并把自己声明为一个COM端口，这样计算机就无法区分。那么当你使用这些USB驱动程序中的一个时，这些USB转串口转换器中的一个，如果你在电脑的控制面板里查看，在端口那边你会看到一个USB串口。假设你有一个自己喜欢的终端程序如putty,我们选择串口，我们可以选择连接到任意线路如COM6，然后设置我们想要的速度，接着它就会进行协商我们就能获取到，不需要流控制，这样我们就可以进行通信了。Arduino软件是通过串口与Windows通信的，所以它需要串口让这个完整的桥接系统保持透明。

FT232R是一款USB转串口UART接口芯片，这里有DTR（数据终端就绪信号），Arduino的复位方式是这样的，它需要一个复位信号。最初他们想，我们怎样才能在进行串口通信的同时，巧妙地实现复位呢？调制解调器接口的一个技巧就是有很多调制解调器线路，数据终端就绪、请求发送、响铃指示、数据集就绪、数据载波检测、允许发送，其中一个就是数据终端就绪，你可以在API中，通过指令将该信号置为低电平或高电平，你能够通过你的串口API来控制它。通常他们会让你与DTR（数据终端就绪）线进行通信。正因为如此，Arduino的设计者们表示，那么，我们可以做的是添加一个API来重置Arduino，你可以对DTR线进行脉冲操作，给他一个脉冲信号，然后我们会在那里放置一个电容器，或者再加个晶体管之类的东西，然后我们会将他连接到复位线上，所以这里有一个0.1微法的电容，直接连接到复位线上。此外，复位线上有一个上拉电阻将其拉高。所以我们只需要TX、RX、DTR这几根线，我们不需要所有其他这些功能了。
